Inputs from Users

Students learned about the input() method in order to provide some sort of interaction with the user as well acquiring the knowledge to capture, store, and retrieve data. One input() method was an opportunity to ask the user which action the user wanted to take from the menu. Dependent on the assignment, the options would be placed in a while loop that when an if or elif statement were met would perform an action. The other input() method used consistently throughout the assignments asked the user to enter a string of text that would  be captured for further use within the program, stored, or retrieved through text input. All had the same type of input, but the lessons usage varied. As you’ll see in Table 1, above, I did not change the method of capturing input from the user. In Assignment05, I wanted to ensure that users were able to easily read the data and had quite a few print statements in my main body which makes it look messy, whereas in Assignment06 it’s cleaned up as we were taught about methods and better organizing our code. Assignment07, for ease I grouped the append() method with the inputs, which I still struggle with as I feel it process the data and should be been placed in the Processor class, but it worked and at the point of creating the working program that’s all that mattered.
